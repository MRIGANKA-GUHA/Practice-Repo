//MONGODB AGGREGATION
$match is the first aggregation we will read in which we give a data to match whether its present or not in the database
$count is the aggregation which count the previous base of aggregation how many is there. It is depend on the previous state
For Example:- [
  {
    $match: {
      isActive: true                  //Here isActive is a variable in db which states whether the user is true or false so we declaring to find how many true
    }
  },                                
  {
    $count: "activeUsers"            //Here we name the count as activeUsers and it count from the previous $match
  }
]

$group another aggregation which is used when we want to calculate or anything to count by categorily grouping.
For Example:- 
[
  {
    $group: {
      _id: null,                         //If we declare null then no category is selected hence every one is in group. If we want to group we can "$nameofentity" in which respect we want to group
      averageAge: {                      //Its the name which name we want to give
        $avg: "$age"                     //Its another function which we used to do average of the group members like here is age
      }
    }
  }
]
One More
List The top 5 favoriteFruit
[
  {
    $group: {
      _id: "$favoriteFruit",                 //Here we group the data in basis of favoriteFruit
      Total: {                               //We make a variable name which will store the result how many is there
        $sum: 1                              //It count 1 by 1 thats why 1 is used.For every data it gets +1 is done
      }
      }
    },
  {
    $sort:{                                  //Then we use $sort for sorting the highest no of participant in favoriteFruit
      Total: -1                              //-1 denote descending order and 1 denotes Ascending order
    }
  },
  {
    $limit: 5                               // It limit how many favoriteFruit will shown 
  }
]
When we have a object inside object Then we have to use this kind of methodology
List the country which have the highest user
[
  {
    $group: {
      _id: "$company.location.country",       //Here we grouping by going inside of the object
      countryUserCount: {
        $sum: 1
      }
    }
  },
  {
    $sort: {
      countryUserCount: -1                 
    }
  },
  {
    $limit: 1
  }
]
When we have a array we do like this way
Average number of tags per user
[
  {
    $unwind: "$tags"                      //It is the opening of the array
  },
  {
    $group: {
      _id: "$_id",                       //Then grouping the users in respect of id
      numberOfTags: {
        $sum: 1
      }
    }
  },
  {
    $group: {
      _id: null,                         //Then we get the avg number of tags
      averageNumberOfTags: {
        $avg: "$numberOfTags"
      }
    }
  }
]

Using Lookup Aggregator
For Example:- 
[
  {
    $lookup: {
      from: "authors",                         //Foreign file
      localField: "author_id",                 //Current file field which to match
      foreignField: "_id",                     //Foreign file field which to match
      as: "author_details"                     //Name in which it have to be save. It will save in array
    }
  },
  {
    $addFields: {                              //It converts the array to list
      author_details: {
        $arrayElemAt: ["$author_details", 0]         
      }
    }
  }
]
